Dictionary (Lug'at) Nima?

Python'da dictionary (qisqacha dict) — bu ma'lumotlarni Key: Value (Kalit: Qiymat) ko'rinishida saqlaydigan ma'lumotlar tuzilmasi. U xuddi real hayotdagi lug'atga o'xshaydi: so'z (kalit) va uning ma'nosi (qiymat).

💡
Data Analyst nuqtai nazaridan:
Dict bu JSON formatining asosi bo'lib, API lar bilan ishlashda, NoSQL bazalarda (MongoDB) va ma'lumotlarni turli xil o'lchovlar (dimensions) bo'yicha guruhlashda eng muhim vosita hisoblanadi.

Dict Yaratish

Dictionary jingalak qavslar {} yoki dict() funksiyasi yordamida yaratiladi.

python 
# Bo'sh dict yaratish
student = {}
student2 = dict()

# Ma'lumot bilan yaratish
user = {
    "id": 101,
    "name": "Saddam",
    "role": "Data Analyst",
    "is_active": True
}
print(user)
⚠️
Muhim eslatma: Dict ichidagi Key (kalit)lar doimo unikal (takrorlanmas) bo'lishi va o'zgarmas (immutable) ma'lumot turlari (string, integer, tuple) bo'lishi shart. List yoki dict ni key sifatida ishlata olmaysiz!

Asosiy Metodlar: .get(), keys(), values(), items()

Elementni olish: to'g'ridan-to'g'ri vs .get()

Dict dan qiymatni olishning 2 usuli bor. Agar kalit mavjud bo'lmasa, to'g'ridan-to'g'ri olish KeyError beradi, .get() esa xatolik bermaydi.

python
sales = {"jan": 1500, "feb": 2000}

# Xavfli usul
print(sales["jan"]) # 1500
# print(sales["mar"]) # KeyError! Dastur qulaydi.

# Xavfsiz usul (Professional)
print(sales.get("mar")) # None qaytaradi
print(sales.get("mar", 0)) # Agar yo'q bo'lsa default 0 qaytaradi

.keys() va .values()

Barcha kalitlarni yoki barcha qiymatlarni ajratib olish uchun.

python
data = {"apple": 50, "banana": 30, "orange": 20}

kalitlar = data.keys()
qiymatlar = data.values()

# Data Analyst logikasi: Jami savdoni hisoblash
total_sales = sum(data.values())
print(f"Jami mevalar soni: {total_sales}") # 100

.items()

Kalit va qiymatni birgalikda Tuple ko'rinishida olish uchun. Asosan for tsiklida aylanib chiqish uchun ishlatiladi.

python
prices = {"USD": 12500, "EUR": 13600}

for currency, rate in prices.items():
    print(f"Valyuta: {currency}, Kurs: {rate}")

Yangilash va O'chirish (Update & Delete)

1. Yangi element qo'shish va qiymatni o'zgartirish ▼

To'g'ridan-to'g'ri kalit orqali yoki .update() metodi orqali.

user = {"name": "Ali"}

# Yangi qo'shish
user["age"] = 25

# Mavjudni o'zgartirish
user["name"] = "Vali"

# update() bilan bir nechta o'zgartirish
user.update({"city": "Tashkent", "age": 26})
2. Elementlarni o'chirish: del, pop, clear ▼ 
data = {"a": 1, "b": 2, "c": 3, "d": 4}

# 1. del - aniq kalit bo'yicha xotiradan o'chiradi
del data["a"]

# 2. pop - o'chiradi va o'chirilgan qiymatni qaytaradi
val = data.pop("b") # val = 2

# 3. popitem - oxirgi qo'shilgan juftlikni o'chiradi (Tuple qaytaradi)
last_item = data.popitem() # ('d', 4)

# 4. clear - dict ni butunlay tozalaydi (bo'm-bo'sh qiladi)
data.clear() # data = {}

Nested Dictionary (Ichma-ich lug'atlar)

Real loyihalarda ma'lumotlar ko'pincha murakkab tuzilishga ega bo'ladi. Dict ichida yana dict saqlash Data Analystlar uchun odatiy hol.

python
company = {
    "IT": {
        "manager": "John",
        "employees": 15
    },
    "HR": {
        "manager": "Anna",
        "employees": 5
    }
}

# Ichki elementga murojaat
it_manager = company["IT"]["manager"]
print(it_manager) # John
🔥
Xatoliklardan qochish: Nested dict larda company["IT"]["budget"] deb murojaat qilsangiz va kalit yo'q bo'lsa xatolik beradi. Professional yo'l: company.get("IT", {}).get("budget", 0)

Data Analyst Logikalari

Haqiqiy analiz stsenariylarida Dict qanday ishlatiladi?

1. O'quvchilar Bahosi Analizi

Max, Min va Average hisoblash.

grades = {"Ali": 85, "Vali": 92, "Gani": 78}

max_g = max(grades.values())
min_g = min(grades.values())
avg_g = sum(grades.values()) / len(grades)

# Top studentni topish logikasi
top_student = [k for k, v in grades.items() if v == max_g][0]

2. KPI Monitoring (Categorization)

Ma'lumotlarni shart bo'yicha guruhlash.

sales = {"Q1": 120, "Q2": 80, "Q3": 150}
status = {"Good": [], "Bad": []}

for q, amount in sales.items():
    if amount >= 100:
        status["Good"].append(q)
    else:
        status["Bad"].append(q)
# Natija: {'Good': ['Q1', 'Q3'], 'Bad': ['Q2']}
Good: Q1, Q3 Bad: Q2

3. Frequency (Chastota) Analizi

Ro'yxatdagi elementlar necha marta qatnashganini sanash.

words = ["data", "ai", "data", "sql", "ai", "data"]
freq = {}

for w in words:
    # get() dagi ajoyib trick
    freq[w] = freq.get(w, 0) + 1
    
# Natija: {'data': 3, 'ai': 2, 'sql': 1}

Data Analyst Tips & Interview Questions

Performance Tip: Dict vs List ▼

Listda element qidirish (in operatori bilan) O(n) vaqt oladi. Ya'ni elementlar qancha ko'p bo'lsa, qidirish shuncha sekinlashadi.

Dictda kalit bo'yicha qidirish Hash Table algoritmi asosida ishlagani uchun O(1) vaqt oladi. Millionlab ma'lumot ichidan keraklisini topish bir zumda amalga oshadi.

Xulosa: Ma'lumotlarni qidirish uchun List emas, Dict foydalaning!

Interview Question 1 ▼

Savol: Ikkita dict ni qanday qilib eng qisqa usulda birlashtirish (merge) mumkin?

Javob: Python 3.9+ da | (pipe) operatori orqali:

d1 = {"a": 1}
d2 = {"b": 2}
d3 = d1 | d2  # natija: {'a': 1, 'b': 2}

Eski versiyalarda: d3 = {**d1, **d2}

Dict Comprehension ▼

Qisqa va professional kod yozish san'ati.

# 1 dan 5 gacha sonlarning kvadratini dict ga joylash
squares = {x: x**2 for x in range(1, 6)}
# natija: {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}
